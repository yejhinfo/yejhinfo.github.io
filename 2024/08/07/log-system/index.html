2.2 成员变量

  • 成员变量：
    • sqlite3* db：SQLite 数据库指针。
    • std::string db_name：数据库文件名。
    • const int MAX_DB_SIZE：最大数据库大小（15MB）

2.3 成员函数

  • 公共成员函数：

    • static LogManager& getInstance(const std::string& db_name = "logs.db")：获取单例实例的静态方法。
    • void log_event(const std::string& event_type, const std::string& event_description, const std::string& source, const std::string& level)：记录日志事件。
    • void query_logs(const std::string& event_type)：查询指定类型的日志记录。

  • 私有成员函数：

    • LogManager(const std::string& db_name)：私有构造函数，初始化数据库连接并创建表和索引。

    • void execute_sql(const std::string& sql)：执行 SQL 语句的辅助函数。

    • int get_db_size()：获取数据库文件大小。

    • void cleanup_logs()：清理旧日志以保持数据库大小在限制内。

总结

  1. 系统架构:

    • 日志源（各模块）
    • 日志收集器（LogManager 类）
    • 日志存储（SQLite 数据库）
    • 日志查询和展示（通过 LogManager 类的查询函数）

  2. 日志记录:

    • 日志字段：时间戳、事件类型、事件描述、来源、日志级别
    • 日志事件：关机、开机、连接器电压设置、连接器子卡检测、电压、电流、功耗、风扇转速、OTA升级、过压、欠压、过流、功耗异常、连接器卡插拔、硬件提供异常信号、FPGA版本加载、FPGA版本清除、复位、子卡复位、全局时钟源变更、可编程时钟频率变更
    示例事件及其说明

    • event_type: Type of the log event

    • event_description: Description of the event

    • source: Module or service generating the log

    • level: Severity level of the log (e.g., DEBUG, INFO, WARN, ERROR)

    1. 关机事件（Shutdown Event）
      • 类型："Shutdown Event"
      • 描述："FpgaShutdown"
      • 来源："Fpga Module"
      • 级别："INFO"
      • 说明：记录系统关机时发生的事件。
    2. 开机事件（Startup Event）
      • 类型："Startup Event"
      • 描述："FpgaStarted"
      • 来源："FpgaModule"
      • 级别："INFO"
      • 说明：记录系统启动时发生的事件。
    3. 连接器电压设置事件（Connector Voltage Setting Event）
      • 类型："Connector Voltage Setting Event"
      • 描述：例如 "Set voltage to 1.5V"
      • 来源："VoltageControlModule"
      • 级别："INFO"
      • 说明：记录设置连接器电压时发生的事件。
    4. 连接器子卡检测事件（Connector Sub-Card Detection Event）
      • 类型："Connector Sub-Card Detection Event"
      • 描述：例如 "Sub-card detected"
      • 来源："DetectionModule"
      • 级别："INFO"
      • 说明：记录检测到连接器子卡时发生的事件。
    5. OTA升级事件（OTA Upgrade Event）
      • 类型："OTA Upgrade Event"
      • 描述：例如 "OTA upgrade to version 2.0"
      • 来源："UpgradeModule"
      • 级别："INFO"
      • 说明：记录 OTA 升级事件。
    6. 过压（Overvoltage）
      • 类型："Overvoltage"
      • 描述：例如 "Voltage exceeds 5.5V"
      • 来源："VoltageMonitorModule"
      • 级别："WARN"
      • 说明：记录电压过高的事件。
    7. 欠压（Undervoltage）
      • 类型："Undervoltage"
      • 描述：例如 "Voltage below 4.5V"
      • 来源："VoltageMonitorModule"
      • 级别："WARN"
      • 说明：记录电压过低的事件。
    8. 过流（Overcurrent）
      • 类型："Overcurrent"
      • 描述：例如 "Current exceeds 3A"
      • 来源："CurrentMonitorModule"
      • 级别："WARN"
      • 说明：记录电流过高的事件。
    9. FPGA版本加载事件（FPGA Version Loading Event）
      • 类型："FPGA Version Loading Event"
      • 描述：例如 "Loaded FPGA version 3.2"
      • 来源："FPGAControlModule"
      • 级别："INFO"
      • 说明：记录FPGA版本加载事件。
    10. FPGA版本清除事件（FPGA Version Clearing Event）
      • 类型："FPGA Version Clearing Event"
      • 描述：例如 "Cleared FPGA version 3.2"
      • 来源："FPGAControlModule"
      • 级别："INFO"
      • 说明：记录FPGA版本清除事件。
    11. 复位事件（Reset Event）
      • 类型："Reset Event"
      • 描述：例如 "System reset"
      • 来源："MainModule"
      • 级别："WARN"
      • 说明：记录系统复位事件。
    12. 子卡复位事件（Sub-Card Reset Event）
      • 类型："Sub-Card Reset Event"
      • 描述：例如 "Reset sub-card"
      • 来源："SubCardControlModule"
      • 级别："WARN"
      • 说明：记录子卡复位事件。
    13. 全局时钟源变更事件（Global Clock Source Change Event）
      • 类型："Global Clock Source Change Event"
      • 描述：例如 "Changed global clock source to X"
      • 来源："ClockControlModule"
      • 级别："INFO"
      • 说明：记录全局时钟源变更事件。
    14. 可编程时钟频率变更事件（Programmable Clock Frequency Change Event）
      • 类型："Programmable Clock Frequency Change Event"
      • 描述：例如 "Changed programmable clock frequency to Y MHz"
      • 来源："ClockControlModule"
      • 级别："INFO"
      • 说明：记录可编程时钟频率变更事件。

  3. 数据库设计:

    • 表结构：日志表（logs）
    • 索引：时间戳索引、事件类型索引

  4. 日志收集器（LogManager 类）:

    • 初始化数据库连接、创建表和索引
    • 记录日志（log_event）
    • 查询日志（query_logs）
